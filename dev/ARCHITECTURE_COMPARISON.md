# Architecture Comparison: Old vs New Codebase

## Overview

The new codebase is larger in raw line count but represents a fundamentally different approach — configuration-driven rather than data-scattered. The old system required a developer to touch 5-6 files and manually repeat layer-specific identifiers across all of them. The new system derives everything from a single config object.

| Metric | Old | New |
|---|---|---|
| Lines per layer (simple) | 63+ across 5-6 files | 33 in 1 file |
| Lines per layer (clickable) | 63-133 across 6 files | ~33 in 1 file |
| Files to touch per new layer | 5-6 | 1 |
| Cross-file sync points per layer | 5+ | 0 |

---

## Adding a Layer

| Scenario | Old | New |
|---|---|---|
| Simple (non-clickable) layer | 63 lines minimum, across 5-6 files | 33 lines, in 1 file |
| Clickable layer (hover, highlight, popup) | 63-133 lines across 6 files | ~33 lines in 1 file |
| Time to add (experienced developer) | 30-60 minutes | 5-10 minutes |
| Time to add (new developer) | Up to half a day | 5-10 minutes |

To add a new layer in the new codebase: copy an existing config entry and swap the values.

---

## Identifier Repetition

In the old codebase, adding one clickable layer required writing layer-specific strings ~37 times across 6 files. Derived IDs like `prev-builds-left`, `prev-builds-right`, `prev-builds-left-highlighted`, and `prev-builds-right-highlighted` had to be written out manually everywhere they were referenced.

In the new codebase, each of the ~10 identifiers is written exactly once, in one file. The engine derives all variants automatically.

| | Old | New |
|---|---|---|
| Identifier instances per layer | ~37 | ~10 |
| Written in how many files | 6 | 1 |
| Derived IDs (e.g. `-highlighted`, `-left`, `-right`) | Written manually everywhere | Generated by the engine |

## Architectural Difference

**Old — data-scattered:**

Layer identity was split across multiple files, each holding a piece of the puzzle:

- `layerSectionData.js` — section/group membership
- `layersList.js` — defined twice (left panel + right panel)
- `clicksHandle.js` — click behavior
- `eventsList.js` — hover and highlight events
- `index.html` — HTML structure
- `modalinfo.js` — info panel content

Any change to a layer required coordinating updates across all of these. A mismatch in any one file silently broke behavior.

**New — data-driven:**

A single config object in `layersList.js` is the source of truth for each layer. The engine reads that config and handles:

- HTML generation
- Checkbox toggles
- Zoom behavior
- Hover and highlight events
- Click handlers
- Info panels

There are zero cross-file sync points. If the config entry is correct, everything works.
